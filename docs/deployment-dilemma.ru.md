## Варианты

### Вариант A: Второй Hetzner как staging

Купить второй Hetzner (EX44 = €39/мес). Полная staging-среда: сборка,
деплой, тест на XRPL testnet, затем ручное продвижение на mainnet.

| + | – |
|---|---|
| Дешевле всего (€39 vs €300 Azure) | Нет SGX — нельзя тестировать DCAP |
| Тот же glibc/arch что mainnet | Ручное продвижение |
| SSH откуда угодно | Нет blue/green |

### Вариант B: Оставить одну Azure VM, остальные деаллоцировать

Оставить одну Azure DCsv3 (node-1) как staging. Деаллоцировать node-2
и node-3. Экономия ~€200/мес.

| + | – |
|---|---|
| Настоящий SGX для staging | Нельзя тестировать 3-нодный multisig |
| €100/мес вместо €300/мес | Одна точка отказа для тестирования |

### Вариант C: Blue/green на самом Hetzner

Два инстанса оркестратора: порт 3000 (mainnet), порт 3001 (staging).
Оба используют один enclave. nginx маршрутизирует. Деплой на 3001,
smoke test, swap.

| + | – |
|---|---|
| Без доп. затрат | Один сервер = общая точка отказа |
| Мгновенный swap через nginx | Данные enclave общие — тест утекает в mainnet |
| Полная mainnet конфигурация | Баг в staging может уронить enclave и mainnet вместе |

**Долгосрочно (при доходе):** Вариант D — Hetzner EX130 с SGX для
mainnet и staging.

## Процедура обновления mainnet на Hetzner

Чеклист безопасного обновления:

1. **Сборка** на Hetzner из git (staging-ветка протестирована)
2. **Снимок** текущего состояния:
   - `cp -r /tmp/perp-9088/ /tmp/perp-9088.backup-$(date +%s)/`
   - `pg_dump perp_dex > /tmp/perp_dex_backup.sql`
3. **Остановка** mainnet: `kill <PID>` (nginx отдаёт 502)
4. **Замена** бинарника: `cp target/release/perp-dex-orchestrator /tmp/perp-9088/`
5. **Запуск** с теми же аргументами
6. **Health check**: `curl http://localhost:3000/v1/health`
7. **Проверка** XRPL mainnet: баланс эскроу не изменился
8. **Откат** если health check не прошёл:
   - `kill <PID>`
   - Восстановить бинарник из backup
   - Перезапуск

**Даунтайм:** ~30 секунд. Приемлемо при текущем трафике (около нуля).

## Чего НЕ делать

- **Никогда** не деплоить непротестированный код на Hetzner mainnet
- **Никогда** не деаллоцировать все Azure VM пока PM ими пользуется
- **Никогда** не запускать `cargo build` на Hetzner в пиковое время
- **Никогда** не перезапускать mainnet без чеклиста бэкапа выше
